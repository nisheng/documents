前端业务及规范

前端业务

业务规范

  1. 所有项目需要把项目的基本信息写在项目的README.md
    1. 必须包含有本地访问、预发访问、线上访问地址
  2. 默认使用打包方式：
npm run dev,使用本地模拟数据
npm run devPre,使用预发接口
npm run pre,预发发布打包
npm run prod;线上发布打包
  1. 模块必须固化，禁止自动升级。
  2. 接口必须有对应的模拟数据。
  3. node_modules禁止提交到git仓库，以免仓库过大。
  4. 最后打包到dist/page(发布到应用服务器)和dist/static(发布到cdn)
